类与对象

一、类

类：一组相关属性和行为的集合。可看作一类事物的模板，使用事物的属性特征和行为特征来描述该事物。

属性：该事物的状态信息
行为：该事物能够做什么

例如：
猫咪类：
属性：名字、体重、年龄、颜色
行为：走、跑、叫、吃饭

二、对象

对象：一类事物的具体体现，对象是类的一个实例，必然具备该类事物的属性和行为

例如：
一只猫
属性：tom、5kg、2 years、yellow
行为：优雅的走，放肆的跑、喵喵的叫、大口的吃饭

三、类与对象的关系

  • 类是对一类事物的描述，是抽象的
  • 对象是一类事物的实例，是具体的
  • 类是对象的模板，对象是类的实体

四、类的定义

类的定义格式

定义类：定义类的实质就是定义类的成员(成员变量、成员方法)

  • 成员变量：与定义变量几乎一致，只是位置发生了改变，定义成员变量的位置在类中，在方法外
  • 成员方法：与定义方法几乎一致，只是把static去掉了

学生实例

定义学生类

定义一个类：用来模拟“学生”事物。其中就有两个组成部分
属性（是什么）：
姓名、年龄等
行为（能做什么）：
吃饭、睡觉、学习
对于在JAVA类当中：
成员变量（属性）：
String name,int age
成员方法（行为）：
注意：成员方法不带有static关键字
public void eat(){}
public void sleep(){}
public void study(){}

使用学生类

通常情况下，一个类不能直接使用，需要根据类创建一个对象，才能使用

  1. 导包：指出需要使用的类在什么位置
    import 包名称.类名称
    例如：import day05.Demo04.Student
    如果和当前类属于同一包的情况，可以省略导包语句不写

  2. 创建：
    格式：
    类名称 对象名 =new 类名称（）;
    例如：Student stu =new Student();

  3. 使用:

    • 两种情况：
      • 使用成员变量：对象名.成员变量名
      • 使用成员方法：对象名.成员方法名（参数）
    • 使用方法概括：用什么，就用对象名称.什么

手机实例

定义手机类

定义一个类，用于模拟手机事物
属性：品牌，价格，颜色
行为：打电话、发短信
则可转化为
成员变量（属性）
String brand;
double price;
String colour;
成员方法（行为）
public void call（String who）{}
public void sendMessage(){}

使用手机类

通常情况下，一个类不能直接使用，需要根据类创建一个对象，才能使用

  1. 导包：指出需要使用的类在什么位置
    import 包名称.类名称
    例如：import day05.Demo04.Phone
    如果和当前类属于同一包的情况，可以省略导包语句不写

  2. 创建：
    格式：
    类名称 对象名 =new 类名称（）;
    例如：Student phone =new Phone();

  3. 使用:

    • 两种情况：
      • 使用成员变量：对象名.成员变量名
      • 使用成员方法：对象名.成员方法名（参数）
    • 使用方法概括：用什么，就用对象名称.什么

注意事项

  1. 成员变量是直接定义在类当中的，在方法外边
  2. 成员方法不要写static关键字
  3. 如果成员变量没有进行赋值，那么将会有一个默认值，规则和数组一样

六、成员变量与局部变量的区别

  1. 定义的位置不一样(重点)

    • 局部变量：在方法的内部
    • 成员变量：在方法的外部，直接写在类当中

  2. 作用范围不一样(重点)

    • 局部变量：只有在方法当中才可以使用，出了方法就不能使用
    • 成员变量：整个类都可以使用

  3. 默认值不一样(重点)

    • 局部变量：如果没有赋值，就无法使用
    • 成员变量：如果没有赋值，会有默认值，规制与数组一样

  4. 内存的位置不一样

    • 局部变量：位于栈内存
    • 成员变量：位于堆内存（成员变量是new出来的，所以成员变量是在堆内存）

  5. 生命周期不同

    • 局部变量：随着方法的进栈而诞生，随着方法的出栈而消失
    • 成员变量：随着对象创建而诞生，随着对象被垃圾回收而消失
      通常情况下局部变量生命周期短一些
